Simple JS slider

Options and Data

All options

Option Default Description
data - jump to data
delay 5000 time each image/group will stay visible, in milliseconds
element - jquery element selector which will be the container
hMargin 10 if view_groups == true, margin between the images in each group, in pixels
height 300 height of the slider, in pixels. if vertical_stretch value is set to true, this value will be increased
load_all true if set to true, images won't be visible until all of them are loaded. otherwise as soon as the first one is loaded it will be shown
max_circles 9999 if the number of images are more than this value, switcher circles won't be shown
resize_if_many false if set to true (and view_groups is true too), images will be resized on the fly. it's useful when the images have different dimensions
show_overlay true to show or not to show the overlay pattern
timeout 1000 time the process of switching from one image/group to another will take, in milliseconds
vPadding 0 vertical padding of the container, in pixels (this only works if view_groups option is set to true, and you may also have to set resize_if_many option to true)
vertical_stretch false if set to true, height of the slider will be increased depending on the height of the screen height. this option comes in hand when the width of a slider is set to 100% of the screen width
view_groups false if set to true, images will be divided into groups and more than one of them will be shown at a time. switcher circles aren't available if using groups
width 900 obviously, width of a slider, in pixels

Data format

  • Data is an array of objects.
  • Each of the objects must have img_src property
  • and may have two other properties - url and title

  • img_src property is of course the source of an image file (image url)

  • if url property is set, the corresponding image will be a link and it's href attribute will be this url

  • title property is a title which will be shown when hovering an image

  • Note that if you use url and/or title, you'll have to set show_overlay option to false. Otherwise title won't be shown and links won't be clickable
var data = [ { "img_src": "img/src.jpg" }, { "img_src": "some/image/file.jpg", "title": "Just a title" }, { "img_src": "path/to/image.jpg", "url": "http://some/url/" }, { "img_src": "http://goo.gl/s6lvL", "url": "http://google.com/", "title": "Google Inc." }, { "img_src": "http://goo.gl/NvmIM", "url": "http://www.ledzeppelin.com/", "title": "Led Zeppelin" } ]

Examples

Using basic options

  • Using slider with just some basic options: width, height, element selector and data.
  • Required options are only element and data.
var options = { width: 700, height: 210, element: $('#s1'), data: [...] };

Controlling the number of circles

  • The number of images in this case is 20.
  • It means that if we change max_circles option to something that's less than 20, for example 9, switcher circles won't appear.
  • Default value for this option is 9999.
var options = { width: 700, height: 210, max_circles: 9, element: $('#s2'), data: [...] };

Different height and width

  • Changed width and height options.
  • Images always match the 100% of the container's width, so the height is proportionally calculated.
  • For example, if you have an image with dimensions of 1000x500 and a slider with width option of 500 and height option of 200, the resized image's size will be 500x250 and the bottom 50 pixels will be overflown.
var options = { width: 600, height: 300, max_circles: 9, element: $('#s3'), data: [...] };

Viewing in groups

  • We're using view_groups option here.
  • It means that images will be divided into groups and instead of switching from one image to another, we'll switch between groups of images.
  • Note that if doing so, switcher circles won't appear.
  • Basically how this works is that a function goes through all the images and adds the width of the current image to the previous sum of widths. If the new sum is greater than the container's width, a new group is created and it starts again. If not, current image is added to the current group and the function continues to the next image.
var options = { width: 700, height: 210, view_groups: true, resize_if_many: true, element: $('#s4'), data: [...] };

Different vertical padding and horizontal margin

  • Using vPadding and hMargin options, which only work when view_groups option is set to true.
  • Default values are 0 for vPadding and 10 for hMargin.
var options = { width: 700, height: 210, vPadding: 10, hMargin: 13, view_groups: true, resize_if_many: true, element: $('#s5'), data: [...] };

Delay and timeout options

  • Using delay and timeout options.
  • delay is the time each image/group will stay visible.
  • timeout is the time the process of switching from one image/group to another will take.
var options = { width: 700, height: 210, timeout: 2000, delay: 3000, element: $('#s6'), data: [...] };
